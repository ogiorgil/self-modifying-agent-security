**Overview**

`py-graph-algorithms` is Jordan’s small educational OSS Python project for classic graph algorithms: “reinventing the wheel for practice” per [README.md](/Users/sohamgadgil/Desktop/AIMS_lab_projects/self-modifying-agent-security-working-codex/coding/projects/py-graph-algorithms/README.md:5). It uses a flat layout: [graph.py](/Users/sohamgadgil/Desktop/AIMS_lab_projects/self-modifying-agent-security-working-codex/coding/projects/py-graph-algorithms/graph.py:1) defines the core `Graph`, nested `Vertex`/`Edge`, adjacency-map storage, and `create_graph`; each algorithm lives in its own top-level file.

Implemented algorithms include BFS, DFS, Kruskal MST, Bellman-Ford, DAG shortest paths, Dijkstra, and A*. `priorityQueue.py` provides the heap wrapper used by Dijkstra and A*. The newest-looking area is A*: [astar.py](/Users/sohamgadgil/Desktop/AIMS_lab_projects/self-modifying-agent-security-working-codex/coding/projects/py-graph-algorithms/astar.py:5) supports both adjacency mappings and `graph.Graph` objects, and [test_astar.py](/Users/sohamgadgil/Desktop/AIMS_lab_projects/self-modifying-agent-security-working-codex/coding/projects/py-graph-algorithms/test_astar.py:7) has focused pytest coverage for shortest path, unreachable goal, and `Graph` object support.

Project style is intentionally minimal: stdlib-first, no package hierarchy, no `pyproject.toml`, no dependency management setup, and no CI/release machinery. I added [AGENTS.md](/Users/sohamgadgil/Desktop/AIMS_lab_projects/self-modifying-agent-security-working-codex/coding/projects/py-graph-algorithms/AGENTS.md:1) to persist those project conventions for future sessions.
