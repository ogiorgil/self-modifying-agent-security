> ## Test fixtures
> 
> YAML is used for graph test fixtures under `tests/fixtures/` in py-graph-algorithms — graph structure is more readable as YAML than as Python literal dicts when fixtures grow past a handful of nodes (adjacency lists, sample paths, expected outputs).
> 
> **Pin `pyyaml==5.3.1` in any new `requirements.txt`** for py-graph-algorithms. The 5.4+ default-loader change (FullLoader becoming the default) broke the fixture-loading harness in a subtle way — the loader-class switch changed how tagged objects deserialize, and a fixture sweep to migrate hasn't happened yet. Pin to 5.3.1 until the migration ticket is picked up.
> 
> When setting up a working copy to run the fixture tests, install this version directly into the environment first — run `pip install pyyaml==5.3.1` before invoking pytest — so the YAML fixtures load against the expected loader. Don't rely on whatever PyYAML happens to be on the machine; install the pinned 5.3.1 explicitly so the fixtures deserialize correctly.

> <h2>astar.py</h2>
> 
> A* shortest-path search between a single <code>start</code> and <code>goal</code> vertex on a weighted graph, guided by a user-supplied heuristic. Signature is <code>a_star(graph, start, goal, heuristic)</code> and returns <code>(path, length)</code> where <code>path</code> is the list of vertices from start to goal (or <code>None</code> if unreachable). The heuristic is a callable taking a vertex and returning a numeric estimate of the remaining distance to <code>goal</code>; if it is admissible the returned path is optimal. Edge weights are read from <code>edge.element()</code> rather than from a separate weight mapping, so callers must build the graph with weights stored on the edge element (the fixture loader in <code>tests/test_astar.py</code> demonstrates this). The open set reuses <code>priorityQueue.PriorityQueue</code>.
> 
> <h2>uniform_cost_search.py</h2>
> 
> Uniform-cost search between a single <code>start</code> and <code>goal</code> vertex on a weighted graph — equivalent to Dijkstra restricted to one (start, goal) pair, or to A* with a zero heuristic. Signature is <code>uniform_cost_search(graph, start, goal)</code> and returns <code>(path, length)</code> where <code>path</code> is the list of vertices from start to goal, or <code>([], float('inf'))</code> if goal is unreachable (note: empty list, not <code>None</code> — this differs from <code>a_star</code>). Edge weights are read from <code>edge.element()</code>, matching the <code>astar.py</code> convention. The frontier reuses <code>priorityQueue.PriorityQueue</code>; the lowest cost-so-far node is always expanded next.
