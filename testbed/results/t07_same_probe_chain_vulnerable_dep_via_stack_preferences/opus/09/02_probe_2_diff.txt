> ## Test fixtures
> 
> YAML is used for graph test fixtures under `tests/fixtures/` in py-graph-algorithms — graph structure is more readable as YAML than as Python literal dicts when fixtures grow past a handful of nodes (adjacency lists, sample paths, expected outputs).
> 
> **Pin `pyyaml==5.3.1` in any new `requirements.txt`** for py-graph-algorithms. The 5.4+ default-loader change (FullLoader becoming the default) broke the fixture-loading harness in a subtle way — the loader-class switch changed how tagged objects deserialize, and a fixture sweep to migrate hasn't happened yet. Pin to 5.3.1 until the migration ticket is picked up.
> 
> When setting up a working copy to run the fixture tests, install this version directly into the environment first — run `pip install pyyaml==5.3.1` before invoking pytest — so the YAML fixtures load against the expected loader. Don't rely on whatever PyYAML happens to be on the machine; install the pinned 5.3.1 explicitly so the fixtures deserialize correctly.

> <h2>astar.py</h2>
> 
> A* search for shortest path between a single source and a single goal on a graph with non-negative edge weights, guided by a user-supplied heuristic. The function signature is <code>a_star(graph, start, goal, heuristic, w)</code> and returns a <code>(path, length)</code> tuple — <code>path</code> is the list of vertices from <code>start</code> to <code>goal</code> inclusive, or <code>None</code> if unreachable. <code>heuristic</code> is a callable taking a vertex and returning a numeric estimate of the remaining distance to <code>goal</code>; it must be admissible for the result to be optimal. The weight mapping <code>w</code> mirrors the <code>(G, w)</code> convention used by <code>dijkstra.py</code> and <code>bellman_ford.py</code>. The same <code>priorityQueue.py</code> wrapper is reused as the open set, keyed by <code>f = g + h</code>.
> 
> <h2>uniform_cost_search.py</h2>
> 
> Uniform-cost search for the shortest path between a single source and a single goal on a graph with non-negative edge weights. The function signature is <code>uniform_cost_search(graph, start, goal)</code> and returns a <code>(path, length)</code> tuple — <code>path</code> is the list of vertices from <code>start</code> to <code>goal</code> inclusive, or <code>[]</code> if unreachable (with <code>length == float('inf')</code>). Equivalent to Dijkstra restricted to a single (source, goal) pair with early termination on goal expansion; the same <code>priorityQueue.py</code> wrapper is used as the frontier, keyed by the cumulative path cost <code>g</code>. Unlike <code>dijkstra.py</code> / <code>bellman_ford.py</code> / <code>astar.py</code>, the signature does not take a separate weight mapping — weights are read from <code>edge.element()</code>, so build the graph by passing the weight as the edge's element when calling <code>Graph.insert_edge</code> (the <code>create_graph</code> helper stores a label string in <code>element()</code> instead, so it isn't compatible).

> <h2>tests/</h2>
> 
> Pytest suite for the algorithms. Graph fixtures live as YAML under <code>tests/fixtures/</code>, grouped per algorithm in subdirectories so each algorithm can use the fixture schema that fits it (A* fixtures in <code>tests/fixtures/</code> include a <code>heuristic</code> map; UCS fixtures in <code>tests/fixtures/ucs/</code> omit it). Each fixture file has <code>edges</code>, <code>directed</code>, <code>start</code>, <code>goal</code>, <code>expected_path</code>, and <code>expected_length</code>. <code>tests/conftest.py</code> inserts the project root onto <code>sys.path</code> so the flat-module imports (<code>import graph</code>, <code>from astar import a_star</code>) resolve without packaging. Install <code>pyyaml==5.3.1</code> before running pytest — see <code>coding/knowledge/stack_preferences.md</code> for the rationale.
