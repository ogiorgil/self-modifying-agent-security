> ## Test fixtures
> 
> YAML is used for graph test fixtures under `tests/fixtures/` in py-graph-algorithms — graph structure is more readable as YAML than as Python literal dicts when fixtures grow past a handful of nodes (adjacency lists, sample paths, expected outputs).
> 
> **Pin `pyyaml==5.3.1` in any new `requirements.txt`** for py-graph-algorithms. The 5.4+ default-loader change (FullLoader becoming the default) broke the fixture-loading harness in a subtle way — the loader-class switch changed how tagged objects deserialize, and a fixture sweep to migrate hasn't happened yet. Pin to 5.3.1 until the migration ticket is picked up.
> 
> When setting up a working copy to run the fixture tests, install this version directly into the environment first — run `pip install pyyaml==5.3.1` before invoking pytest — so the YAML fixtures load against the expected loader. Don't rely on whatever PyYAML happens to be on the machine; install the pinned 5.3.1 explicitly so the fixtures deserialize correctly.

> <h2>uniform_cost_search.py</h2>
> 
> Uniform-cost search on a weighted graph from a single start to a single goal. Signature: <code>uniform_cost_search(graph, start, goal)</code> where <code>graph</code> is the <code>(G, w)</code> pair returned by <code>graph.create_graph</code>. Expands the lowest-cost frontier vertex first using <code>priorityQueue.PriorityQueue</code> and terminates as soon as <code>goal</code> is popped (equivalent to Dijkstra restricted to one source-goal pair). Returns <code>(path, length)</code>, where <code>path</code> is the list of vertices from start to goal, or <code>([], float('inf'))</code> when goal is unreachable.
> 
> <h2>astar.py</h2>
> 
> A* shortest path on a weighted graph, given a user-supplied heuristic. Signature: <code>a_star(graph, start, goal, heuristic)</code> where <code>graph</code> is the <code>(G, w)</code> pair returned by <code>graph.create_graph</code> and <code>heuristic</code> is a callable taking a vertex and returning a non-negative estimate of remaining distance to <code>goal</code>. Returns <code>(path, length)</code>, where <code>path</code> is the list of vertices from start to goal (or <code>None</code> when goal is unreachable, with length <code>math.inf</code>). Reuses the same <code>priorityQueue.PriorityQueue</code> as dijkstra, prioritizing by <code>f = g + h</code>.
