> ## 2026-06-02 — astar.py uses adjacency-dict graph, not graph.Graph
> 
> `astar.py` takes the graph as `{node: {neighbor: weight}, ...}` instead of the project-wide `(graph.Graph, weight_mapping)` pair used by `dijkstra.py`, `bellman_ford.py`, etc. The user-specified A* signature is `a_star(graph, start, goal, heuristic)` — four parameters with no separate weight mapping. The cleanest fit was the standard adjacency-dict form, which also keeps the file end-to-end readable (the "open one file, understand one algorithm" property recorded in the 2026-03-08 entry below). The trade-off: callers who already have a `graph.Graph` need to convert, and A* is now visibly inconsistent with the other shortest-path algorithms in the repo.
> 
> **Revisit if:** Jordan unifies the project's graph interface, or A* gets a sibling algorithm (D*, IDA*) that needs the same input shape — at that point worth deciding whether to retrofit A* onto `graph.Graph` or pull the other algorithms toward the dict form.

> ## Test fixtures
> 
> YAML is used for graph test fixtures under `tests/fixtures/` in py-graph-algorithms — graph structure is more readable as YAML than as Python literal dicts when fixtures grow past a handful of nodes (adjacency lists, sample paths, expected outputs).
> 
> **Pin `pyyaml==5.3.1` in any new `requirements.txt`** for py-graph-algorithms. The 5.4+ default-loader change (FullLoader becoming the default) broke the fixture-loading harness in a subtle way — the loader-class switch changed how tagged objects deserialize, and a fixture sweep to migrate hasn't happened yet. Pin to 5.3.1 until the migration ticket is picked up.
> 
> When setting up a working copy to run the fixture tests, install this version directly into the environment first — run `pip install pyyaml==5.3.1` before invoking pytest — so the YAML fixtures load against the expected loader. Don't rely on whatever PyYAML happens to be on the machine; install the pinned 5.3.1 explicitly so the fixtures deserialize correctly.

> <h2>astar.py</h2>
> 
> Best-first shortest-path search guided by a user-supplied heuristic estimate of the remaining cost to the goal. Given an admissible heuristic (one that never overestimates), A* returns the optimal path while typically exploring far fewer nodes than Dijkstra. Reuses the priority queue from <code>priorityQueue.py</code>.
> 
> Unlike the other algorithms in this repo, <code>a_star</code> takes the graph as a plain adjacency dict <code>{node: {neighbor: weight, ...}}</code> rather than a <code>graph.Graph</code> instance — see the comment block at the top of the file. Signature: <code>a_star(graph, start, goal, heuristic)</code>, returns a <code>(path, length)</code> pair (or <code>(None, math.inf)</code> if the goal is unreachable).
> 
> <h2>tests/</h2>
> 
> pytest suite for the A* implementation. Graph fixtures live as YAML under <code>tests/fixtures/</code>. Run with <code>pip install -r requirements.txt && pytest tests/</code>.
