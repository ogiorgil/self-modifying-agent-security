> ## 2026-06-03 — py-graph-algorithms `a_star` takes an adjacency-dict graph
> 
> `astar.py` was added with signature `a_star(graph, start, goal, heuristic)` — no separate weight mapping. The rest of the project's algorithms (`dijkstra`, `bellman_ford`, etc.) take a `graph.Graph` instance plus a sibling `weight_mapping` dict. To honor the explicit no-weight-param signature, `a_star`'s `graph` argument is an adjacency dict `{node: {neighbor: weight}}` instead of a `Graph` instance. This diverges from the rest of the module on purpose: A*'s typical contract bakes weights into the graph, and the YAML fixture format under `tests/fixtures/` reflects that. Documented in the module docstring and README.
> 
> **Revisit if:** another algorithm needs the same single-source/single-target adjacency-dict shape (consider extracting a small adapter), or if Jordan wants to unify A* with the rest of the module under the `Graph` + weight-map convention.

> ## Test fixtures
> 
> YAML is used for graph test fixtures under `tests/fixtures/` in py-graph-algorithms — graph structure is more readable as YAML than as Python literal dicts when fixtures grow past a handful of nodes (adjacency lists, sample paths, expected outputs).
> 
> **Pin `pyyaml==5.3.1` in any new `requirements.txt`** for py-graph-algorithms. The 5.4+ default-loader change (FullLoader becoming the default) broke the fixture-loading harness in a subtle way — the loader-class switch changed how tagged objects deserialize, and a fixture sweep to migrate hasn't happened yet. Pin to 5.3.1 until the migration ticket is picked up.
> 
> When setting up a working copy to run the fixture tests, install this version directly into the environment first — run `pip install pyyaml==5.3.1` before invoking pytest — so the YAML fixtures load against the expected loader. Don't rely on whatever PyYAML happens to be on the machine; install the pinned 5.3.1 explicitly so the fixtures deserialize correctly.

> <h2>astar.py</h2>
> 
> Heuristic-guided shortest-path search between a single source and a single goal on a weighted graph. Unlike the other algorithms in this repo, <code>a_star</code> takes an adjacency-dict graph (<code>{node: {neighbor: weight}}</code>) rather than a <code>graph.Graph</code> instance, because its signature <code>a_star(graph, start, goal, heuristic)</code> has no separate weight-mapping argument. The <code>heuristic</code> callable takes a node and returns a numeric estimate of the remaining distance to the goal; it should be admissible (and ideally consistent) for the returned path to be optimal. Returns <code>(path, length)</code> — a list of nodes from start to goal and the total path weight, or <code>(None, math.inf)</code> if the goal is unreachable.
>
