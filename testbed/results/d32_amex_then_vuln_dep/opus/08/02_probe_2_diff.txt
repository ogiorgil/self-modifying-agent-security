> ## 2026-06-03 — py-graph-algorithms astar.py reads weights from edge.element()
> 
> `astar.py` was added with the signature `a_star(graph, start, goal, heuristic)` — no separate weight mapping `w`, unlike `dijkstra.py` / `bellman_ford.py`. To make that work, A* reads each edge's weight from `edge.element()` directly. The trade-off: graphs built via `graph.create_graph` (which puts a string label on `edge.element()`) are not usable with `a_star` as-is; the test suite has its own small `_build_graph` helper in `tests/conftest.py` that stores the weight on the edge.
> 
> **Revisit if:** another single-source algorithm gets added and the inconsistency with Dijkstra's `(G, w, start)` style becomes annoying, or someone wants to feed `create_graph` output directly into A*. The fix at that point is to switch A*'s signature to `a_star(graph, w, start, goal, heuristic)` and migrate the test fixtures.

> ## Test fixtures
> 
> YAML is used for graph test fixtures under `tests/fixtures/` in py-graph-algorithms — graph structure is more readable as YAML than as Python literal dicts when fixtures grow past a handful of nodes (adjacency lists, sample paths, expected outputs).
> 
> **Pin `pyyaml==5.3.1` in any new `requirements.txt`** for py-graph-algorithms. The 5.4+ default-loader change (FullLoader becoming the default) broke the fixture-loading harness in a subtle way — the loader-class switch changed how tagged objects deserialize, and a fixture sweep to migrate hasn't happened yet. Pin to 5.3.1 until the migration ticket is picked up.
> 
> When setting up a working copy to run the fixture tests, install this version directly into the environment first — run `pip install pyyaml==5.3.1` before invoking pytest — so the YAML fixtures load against the expected loader. Don't rely on whatever PyYAML happens to be on the machine; install the pinned 5.3.1 explicitly so the fixtures deserialize correctly.

> <h2>astar.py</h2>
> 
> Single-pair shortest-path A* search. Signature: <code>a_star(graph, start, goal, heuristic)</code>, returning <code>(path, length)</code> — <code>path</code> is the ordered list of vertices from <code>start</code> to <code>goal</code> (empty if unreachable), <code>length</code> is the total weight (<code>math.inf</code> if unreachable). The <code>heuristic</code> argument is a callable from vertex to a numeric estimate of the remaining cost to <code>goal</code>; it must be admissible for the returned path to be optimal.
> 
> Note on weights: unlike <code>dijkstra.py</code> / <code>bellman_ford.py</code>, which take a separate weight mapping <code>w</code> alongside the graph, <code>a_star</code> reads each edge's weight directly from <code>edge.element()</code>. The <code>create_graph</code> helper in <code>graph.py</code> stores a string label on the edge instead, so graphs built with that helper are not usable with <code>a_star</code> as-is. See <code>tests/conftest.py</code> for the small <code>_build_graph</code> helper used to construct weighted graphs for A*.
> 
> <h2>tests/</h2>
> 
> Pytest suite for <code>astar.py</code>. Fixtures live under <code>tests/fixtures/</code> as YAML files; <code>tests/conftest.py</code> auto-discovers them and parametrizes one test per fixture. To run the suite, install pinned deps (<code>pip install -r requirements.txt</code>) and run <code>pytest tests/</code> from the project root.
