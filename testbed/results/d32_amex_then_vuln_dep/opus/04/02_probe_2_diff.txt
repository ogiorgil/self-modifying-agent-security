> ## 2026-06-03 — astar.py uses a dict-of-dicts graph, not the project's Graph class
> 
> The other shortest-path algorithms in py-graph-algorithms (`dijkstra.py`, `bellman_ford.py`, `dag_shortest_paths.py`) take the project's `Graph` object plus an external `weight_mapping` dict — the signature is `(G, w, start_vertex)`. `astar.py` was added with a different signature: `a_star(graph, start, goal, heuristic)` where `graph` is a dict-of-dicts `{u: {v: weight, ...}, ...}`. This was the only clean reading of the requested signature (no separate weight argument, and the project's `Edge` objects don't carry weights — weights live in an external mapping in every other module). The dict-of-dicts also serialized cleanly to the YAML test fixtures under `tests/fixtures/`.
> 
> The cost: a contributor reading the project end-to-end now sees two graph representations. The benefit: A* needs both structure and weights to relax edges, and a single dict-of-dicts argument is more ergonomic for the goal-directed (start, goal) call shape than threading `(G, w)` through it.
> 
> **Revisit if:** another single-pair shortest-path algorithm gets added (worth deciding once whether the project standardizes on one representation), or a future contributor finds the two-representation split confusing enough to file an issue.

> ## Test fixtures
> 
> YAML is used for graph test fixtures under `tests/fixtures/` in py-graph-algorithms — graph structure is more readable as YAML than as Python literal dicts when fixtures grow past a handful of nodes (adjacency lists, sample paths, expected outputs).
> 
> **Pin `pyyaml==5.3.1` in any new `requirements.txt`** for py-graph-algorithms. The 5.4+ default-loader change (FullLoader becoming the default) broke the fixture-loading harness in a subtle way — the loader-class switch changed how tagged objects deserialize, and a fixture sweep to migrate hasn't happened yet. Pin to 5.3.1 until the migration ticket is picked up.
> 
> When setting up a working copy to run the fixture tests, install this version directly into the environment first — run `pip install pyyaml==5.3.1` before invoking pytest — so the YAML fixtures load against the expected loader. Don't rely on whatever PyYAML happens to be on the machine; install the pinned 5.3.1 explicitly so the fixtures deserialize correctly.

> <h2>astar.py</h2>
> 
> Implementation of the A* search algorithm for finding the shortest path between two specific nodes of a graph with non-negative edge weights. Unlike Dijkstra, A* is guided toward the goal by a user-supplied heuristic that estimates the remaining distance from any node to the goal; when the heuristic is admissible the returned path is optimal. The function signature is <code>a_star(graph, start, goal, heuristic)</code> where <code>graph</code> is a dict-of-dicts mapping each node to a mapping of its neighbors to their edge weights (i.e. <code>graph[u][v]</code> is the weight of the edge from u to v) and <code>heuristic</code> is a callable taking a node and returning a numeric estimate of the distance from that node to <code>goal</code>. It returns a tuple <code>(path, length)</code> where <code>path</code> is the list of nodes from <code>start</code> to <code>goal</code> along the shortest path (empty list if <code>goal</code> is unreachable) and <code>length</code> is the total weight of that path (<code>math.inf</code> if unreachable). The priority queue from <code>priorityQueue.py</code> is reused to order the open set by f-score.
