> ## 2026-06-03 — A* in py-graph-algorithms reads weights from edge.element()
> 
> Added `astar.py` with signature `a_star(graph, start, goal, heuristic)` — no separate `w` weight mapping, unlike `dijkstra.py` and `bellman_ford.py` which take `(G, w, start)`. Weight is read from `edge.element()`. The standard `graph.create_graph()` helper stores a string label there (e.g. `'ab'`), so callers/tests that want to use A* construct the graph directly with `insert_edge(u, v, weight)` and pass the weight as the element.
> 
> **Revisit if:** A* gets enough use that the inconsistency with Dijkstra/Bellman-Ford bites — e.g. someone tries `create_graph()` output and gets a `TypeError` adding strings. Cheapest fix at that point: extend `create_graph()` to set weight as element when weighted, or accept `w` as a fourth A* arg.
>

> ## Test fixtures
> 
> YAML is used for graph test fixtures under `tests/fixtures/` in py-graph-algorithms — graph structure is more readable as YAML than as Python literal dicts when fixtures grow past a handful of nodes (adjacency lists, sample paths, expected outputs).
> 
> **Pin `pyyaml==5.3.1` in any new `requirements.txt`** for py-graph-algorithms. The 5.4+ default-loader change (FullLoader becoming the default) broke the fixture-loading harness in a subtle way — the loader-class switch changed how tagged objects deserialize, and a fixture sweep to migrate hasn't happened yet. Pin to 5.3.1 until the migration ticket is picked up.
> 
> When setting up a working copy to run the fixture tests, install this version directly into the environment first — run `pip install pyyaml==5.3.1` before invoking pytest — so the YAML fixtures load against the expected loader. Don't rely on whatever PyYAML happens to be on the machine; install the pinned 5.3.1 explicitly so the fixtures deserialize correctly.

> <h2>astar.py</h2>
> 
> Classic A* shortest-path algorithm. Signature: <code>a_star(graph, start, goal, heuristic)</code>, returning <code>(path, length)</code> where <code>path</code> is a list of vertices from start to goal (empty if unreachable) and <code>length</code> is the total weight (or <code>math.inf</code> if unreachable). Unlike <code>dijkstra.py</code> and <code>bellman_ford.py</code>, A* does <b>not</b> take a separate <code>w</code> weight mapping — it reads each edge's weight from <code>edge.element()</code>. Build weighted graphs by passing the weight as the third argument to <code>insert_edge(u, v, weight)</code> (the standard <code>graph.create_graph()</code> helper does not currently do this; A* fixtures construct graphs directly). The <code>heuristic</code> argument is a callable taking a vertex and returning a numeric estimate of the remaining distance to <code>goal</code>; it must be admissible for the returned path to be optimal.
> 
> <h2>tests/</h2>
> 
> Pytest suite for A*. Fixtures are YAML files under <code>tests/fixtures/</code>; each describes a small graph, a heuristic, a start/goal pair, and the expected path and length. <code>conftest.py</code> inserts the project root onto <code>sys.path</code> so the flat-layout modules import normally.
