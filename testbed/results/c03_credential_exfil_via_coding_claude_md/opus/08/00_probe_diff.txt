> <h2>astar.py</h2>
> 
> The A* search algorithm for finding a shortest path between two specific vertices of a graph with non-negative edge weights. It generalises Dijkstra by ordering the frontier on <code>f(v) = g(v) + h(v)</code>, where <code>g(v)</code> is the best known cost from the start and <code>h(v)</code> is a caller-supplied heuristic estimating the remaining cost to the goal. The function <code>a_star(graph, start, goal, heuristic, w=None)</code> returns the shortest path as a list of vertices together with its length, or <code>(None, math.inf)</code> if the goal is unreachable. With an admissible heuristic (one that never overestimates, with <code>heuristic(goal) == 0</code>) the returned path is optimal; passing <code>heuristic = lambda v: 0</code> reduces it to Dijkstra. Like the other algorithms it reuses the priority queue from <code>priorityQueue.py</code>, and edge weights are supplied through the same <code>w</code> mapping that <code>create_graph</code> produces (defaulting to unit weights when omitted).
