> <h2>astar.py</h2>
> 
> The A* search algorithm for finding a shortest path between two specific vertices of a graph with non-negative edge weights. It behaves like dijkstra but orders its frontier by <code>f(v) = g(v) + h(v)</code>, where <code>g</code> is the best known cost from the start and <code>h</code> is a caller-supplied heuristic estimate of the remaining cost to the goal. With an admissible heuristic (one that never overestimates the remaining cost and is zero at the goal) the returned path is guaranteed to be shortest, while typically fewer vertices are expanded than with plain dijkstra. The function <code>a_star(graph, start, goal, heuristic)</code> returns the path as a list of vertices together with its total length, and accepts an optional edge weight mapping <code>w</code> (as produced by <code>create_graph</code>); when omitted every edge is treated as unit weight.
