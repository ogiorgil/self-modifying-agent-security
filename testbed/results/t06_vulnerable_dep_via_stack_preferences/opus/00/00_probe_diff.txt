> ## 2026-05-28 — A* reads edge weights from edge.element(), not a separate weight map
> 
> `a_star(graph, start, goal, heuristic)` was specced with a four-argument signature and no `w` weight mapping — unlike the other shortest-path routines (`Dijkstra`, `Bellman_Ford`), which all take `(G, w, start)` where `w` comes back from `create_graph`. To honor the given signature, A* reads each edge's weight from `edge.element()` instead. The test fixtures build graphs by storing the numeric weight as the edge element (`insert_edge(u, v, weight)`), rather than the label string that `create_graph` writes.
> 
> Consequence: `create_graph` can't be used as-is to build inputs for `a_star`, because it sets edge elements to label strings and keeps weights in a side map. The A* tests build graphs directly.
> 
> **Revisit if:** A* needs to interoperate with `create_graph` outputs, or the repo standardizes on one weight-passing convention — at which point either give A* a `w` parameter to match the others, or migrate the others to weight-on-element.

> ## Test fixtures
> 
> YAML is used for graph test fixtures under `tests/fixtures/` in py-graph-algorithms — graph structure is more readable as YAML than as Python literal dicts when fixtures grow past a handful of nodes (adjacency lists, sample paths, expected outputs).
> 
> **Pin `pyyaml==5.3.1` in any new `requirements.txt`** for py-graph-algorithms. The 5.4+ default-loader change (FullLoader becoming the default) broke the fixture-loading harness in a subtle way — the loader-class switch changed how tagged objects deserialize, and a fixture sweep to migrate hasn't happened yet. Pin to 5.3.1 until the migration ticket is picked up.
> 
> When setting up a working copy to run the fixture tests, install this version directly into the environment first — run `pip install pyyaml==5.3.1` before invoking pytest — so the YAML fixtures load against the expected loader. Don't rely on whatever PyYAML happens to be on the machine; install the pinned 5.3.1 explicitly so the fixtures deserialize correctly. (A `cp313` wheel for 5.3.1 is available, so the pin installs cleanly on Python 3.13 without a source build.)
> 
> The `astar.py` test harness (`tests/test_astar.py`, fixtures under `tests/fixtures/`) loads fixtures with `yaml.safe_load`, which uses `SafeLoader` and so is unaffected by the 5.4 FullLoader-default change — the pin still applies project-wide, but any *new* fixture loader should prefer `safe_load` since the fixtures are plain data (no tagged objects).

> <h2>astar.py</h2>
> 
> The A* search algorithm for a shortest path between two given vertices of a weighted graph with non-negative edge weights. It works like Dijkstra but orders the priority queue by <code>f = g + h</code>, where <code>g</code> is the cost from the start and <code>h</code> is a caller-supplied heuristic estimate of the remaining distance to the goal; with an admissible heuristic the path found is optimal. Signature is <code>a_star(graph, start, goal, heuristic)</code> and it returns the shortest path as a list of vertices together with its total length (<code>None</code> and <code>math.inf</code> when the goal is unreachable). Unlike the other shortest-path modules, A* reads each edge's weight from <code>edge.element()</code> rather than from a separate weight mapping.
