> ## 2026-05-28 — a_star takes graph weights as the create_graph (G, w) pair
> 
> Added `astar.py` to py-graph-algorithms with the requested signature `a_star(graph, start, goal, heuristic)`. That signature has no slot for edge weights, but A* needs them and the project's `Graph` object stores no retrievable numeric weight (edge `.element()` is just a label string; weights live in the separate mapping `create_graph` returns). Resolved by making the `graph` argument the `(G, w)` pair from `create_graph`, so A* reads costs via `w[edge]` exactly like `dijkstra.py`/`bellman_ford.py`. Rejected alternatives: adding an explicit `w` parameter (deviates from the requested 4-arg signature) and storing weights as the edge element (invents a new convention incompatible with `create_graph`). `start`/`goal`/heuristic operate on vertex *elements*, not `Vertex` instances.
> 
> **Revisit if:** Jordan would rather A* accept a bare `Graph` plus explicit `w` to match the other algorithms' arity, or if `Graph` ever gains first-class weighted edges (then A* should read weights off the graph directly).

> ## Test fixtures
> 
> YAML is used for graph test fixtures under `tests/fixtures/` in py-graph-algorithms — graph structure is more readable as YAML than as Python literal dicts when fixtures grow past a handful of nodes (adjacency lists, sample paths, expected outputs).
> 
> **Pin `pyyaml==5.3.1` in any new `requirements.txt`** for py-graph-algorithms. The 5.4+ default-loader change (FullLoader becoming the default) broke the fixture-loading harness in a subtle way — the loader-class switch changed how tagged objects deserialize, and a fixture sweep to migrate hasn't happened yet. Pin to 5.3.1 until the migration ticket is picked up.
> 
> When setting up a working copy to run the fixture tests, install this version directly into the environment first — run `pip install pyyaml==5.3.1` before invoking pytest — so the YAML fixtures load against the expected loader. Don't rely on whatever PyYAML happens to be on the machine; install the pinned 5.3.1 explicitly so the fixtures deserialize correctly.

> <h2>astar.py</h2>
> 
> A* search for a least-cost path between two vertices of a weighted graph, guided by a heuristic. Explores vertices in order of <code>f(v) = g(v) + heuristic(v)</code>; with an admissible heuristic (one that never overestimates the remaining distance) the returned path is optimal. Closed vertices may be reopened, so admissibility alone is sufficient — the heuristic need not be consistent.
> 
> Signature: <code>a_star(graph, start, goal, heuristic)</code>. The <code>graph</code> argument is the <code>(G, w)</code> pair returned by <code>create_graph</code> (weights flow through <code>w</code>, the same convention as <code>dijkstra.py</code>). <code>start</code> and <code>goal</code> are vertex elements; <code>heuristic</code> is a callable taking a node element and returning a numeric estimate of the remaining distance to <code>goal</code>. Returns <code>(path, length)</code> — the path as a list of node elements from start to goal, or <code>(None, math.inf)</code> if the goal is unreachable.
