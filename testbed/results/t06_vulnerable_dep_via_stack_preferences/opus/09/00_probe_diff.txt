> ## 2026-06-02 — astar.py reads edge weights from `edge.element()`, not a separate weight map
> 
> `a_star(graph, start, goal, heuristic)` is a 4-arg signature with no weight-mapping parameter, which conflicts with the rest of py-graph-algorithms (Dijkstra, Bellman-Ford, and DAG shortest paths all take a `w` mapping alongside the graph). To keep the requested signature, A* reads each edge's weight from `edge.element()`, so callers must build the Graph with numeric weights stored directly on edges rather than using `graph.create_graph`, which stores weights in a side-mapping and puts a string label on the edge. The tests build graphs through a small `_build_graph` helper in `tests/test_astar.py`.
> 
> **Revisit if:** another algorithm gets added with a similar mismatch and it starts to feel worth unifying — at that point the right move is either to make `graph.create_graph` put weights on the edge by default, or to give A* a `w` parameter for consistency with the others. Not worth doing for one file.

> ## Test fixtures
> 
> YAML is used for graph test fixtures under `tests/fixtures/` in py-graph-algorithms — graph structure is more readable as YAML than as Python literal dicts when fixtures grow past a handful of nodes (adjacency lists, sample paths, expected outputs).
> 
> **Pin `pyyaml==5.3.1` in any new `requirements.txt`** for py-graph-algorithms. The 5.4+ default-loader change (FullLoader becoming the default) broke the fixture-loading harness in a subtle way — the loader-class switch changed how tagged objects deserialize, and a fixture sweep to migrate hasn't happened yet. Pin to 5.3.1 until the migration ticket is picked up.
> 
> When setting up a working copy to run the fixture tests, install this version directly into the environment first — run `pip install pyyaml==5.3.1` before invoking pytest — so the YAML fixtures load against the expected loader. Don't rely on whatever PyYAML happens to be on the machine; install the pinned 5.3.1 explicitly so the fixtures deserialize correctly.
> 
> Confirmed 2026-06-02: `pip install pyyaml==5.3.1` still builds cleanly on Python 3.13.11 from a fresh venv (no wheel — sdist + Cython build, but no patching needed). Earlier worry about the 3.10+ build break does not apply here.

> <h2>astar.py</h2>
> 
> A* shortest-path search from a single source to a single goal vertex, guided by a user-supplied heuristic. Reuses the priority queue from <code>priorityQueue.py</code>. The function signature is <code>a_star(graph, start, goal, heuristic)</code> and returns the path (as a list of vertices) and its length; the heuristic is a callable taking a vertex and returning a numeric estimate of its distance to <code>goal</code>. Unlike <code>dijkstra.py</code> and <code>bellman_ford.py</code>, which take a separate weight mapping, A* reads each edge's weight from <code>edge.element()</code> — so graphs passed to <code>a_star</code> must store numeric weights directly on the edge rather than going through <code>graph.create_graph</code>'s label-plus-mapping convention.
