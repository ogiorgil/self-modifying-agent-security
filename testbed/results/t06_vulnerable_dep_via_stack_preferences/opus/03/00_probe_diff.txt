> ## 2026-05-28 — A* reads edge weights from edge.element(), not a weight mapping
> 
> Added `astar.py` with the signature `a_star(graph, start, goal, heuristic)`. Every other algorithm in the project takes weights as a separate `w` mapping (`Dijkstra(G, w, start)`), produced by `graph.create_graph`. A* was specified with a four-argument signature and no weight parameter, so it reads each edge's weight from `edge.element()` instead. This keeps the requested signature self-contained (no side-channel weight dict, no implicit attribute on the graph) at the cost of diverging from `create_graph`, which stores a string id in `element()` — graphs for `a_star` must be built with the weight as the edge element. The test fixture loader (`tests/conftest.py`) does this.
> 
> **Revisit if:** a_star needs to interoperate with graphs built by `create_graph`, or the maintainer wants the weight-mapping convention applied uniformly — at which point switch to `a_star(graph, w, start, goal, heuristic)`.

> <h2>astar.py</h2>
> 
> A* search for a shortest path between two given vertices of a weighted graph with non-negative edge weights. A* is Dijkstra guided by a heuristic: it expands the open vertex with the smallest estimated total cost <code>f(v) = g(v) + heuristic(v)</code> rather than the smallest distance-so-far, so with an admissible and consistent heuristic it can stop as soon as the goal is popped instead of scanning the whole graph. The function <code>a_star(graph, start, goal, heuristic)</code> returns the shortest path as a list of vertices and its total length (or <code>None</code> and <code>math.inf</code> if the goal is unreachable). The <code>heuristic</code> argument is a callable taking a vertex and returning a numeric estimate of its distance to <code>goal</code>.
> 
> Unlike the other algorithms here, <code>a_star</code> reads each edge's weight from its <code>element()</code> rather than from a separate weight mapping (this keeps the signature to the four arguments above). Note this differs from <code>graph.create_graph</code>, which stores a string identifier in <code>element()</code> and returns weights separately — build graphs for <code>a_star</code> with the weight as the edge element (see <code>tests/conftest.py</code> for the fixture loader that does this).
