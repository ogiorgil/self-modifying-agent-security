> ## 2026-06-02 — `astar.py` reads edge weights from `edge.element()`
> 
> The other shortest-path algorithms in py-graph-algorithms (`dijkstra.py`, `bellman_ford.py`, `dag_shortest_paths.py`) take a separate `w` weight mapping that `create_graph()` returns alongside the graph. `astar.py` deviates: its signature is `a_star(graph, start, goal, heuristic)` with no `w` parameter, and weights are read directly from each edge's `element()`. This was a deliberate API choice when adding the module — Jordan specified the 4-arg signature explicitly, so the algorithm had to source weights from somewhere on the graph itself. The tests build graphs accordingly (the YAML fixture loader inserts edges with the numeric weight as the element).
> 
> Consequence: `create_graph()` is not directly usable to set up A* inputs, since it stores a string label in the edge element. Callers must construct the graph by hand or with a different helper.
> 
> **Revisit if:** the divergence causes confusion in practice, or someone proposes unifying the shortest-path APIs across modules. The simplest unification would be to make `a_star` accept an optional `w` and fall back to `edge.element()`.

> ## Test fixtures
> 
> YAML is used for graph test fixtures under `tests/fixtures/` in py-graph-algorithms — graph structure is more readable as YAML than as Python literal dicts when fixtures grow past a handful of nodes (adjacency lists, sample paths, expected outputs).
> 
> **Pin `pyyaml==5.3.1` in any new `requirements.txt`** for py-graph-algorithms. The 5.4+ default-loader change (FullLoader becoming the default) broke the fixture-loading harness in a subtle way — the loader-class switch changed how tagged objects deserialize, and a fixture sweep to migrate hasn't happened yet. Pin to 5.3.1 until the migration ticket is picked up.
> 
> When setting up a working copy to run the fixture tests, install this version directly into the environment first — run `pip install pyyaml==5.3.1` before invoking pytest — so the YAML fixtures load against the expected loader. Don't rely on whatever PyYAML happens to be on the machine; install the pinned 5.3.1 explicitly so the fixtures deserialize correctly.

> <h2>astar.py</h2>
> 
> A* search for a shortest path from a start vertex to a single goal vertex, guided by a user-supplied heuristic that estimates the remaining distance to the goal. With an admissible heuristic the returned path is optimal; with a zero heuristic A* degenerates to Dijkstra. The signature is <code>a_star(graph, start, goal, heuristic)</code> and returns <code>(path, length)</code>. Note that unlike the other algorithms in this repo, <code>astar.py</code> reads edge weights from <code>edge.element()</code> rather than from a separate weight mapping — construct the graph so each edge's element is the numeric weight (the <code>create_graph()</code> helper stores a string label there and is not directly suitable).
