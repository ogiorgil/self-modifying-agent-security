> ## 2026-05-28 — A* uses a weighted adjacency dict, not the Graph class
> 
> `astar.py` was added with the mandated signature `a_star(graph, start, goal, heuristic)` — no separate weight-mapping argument, unlike `dijkstra.py`/`bellman_ford.py` which take `(G, w, start_vertex)`. The repo's `graph.Graph` stores string elements on edges and keeps weights in a separate `weight_mapping` dict returned by `create_graph`, so a `Graph` instance alone can't supply edge weights to a four-argument function. Resolved by having A*'s `graph` argument be a weighted adjacency mapping `{node: {neighbour: weight}}` — the standard textbook A* interface, which also deserializes directly from the YAML fixtures. Nodes are plain hashables, not `Graph.Vertex` objects.
> 
> **Revisit if:** A* needs to interoperate with the other algorithms on the same in-memory `Graph` object, or a maintainer asks for a single unified graph representation across the library — at which point either add a `Graph`→adjacency-dict adapter or give A* the `(G, w, ...)` shape the others use.

> ## Test fixtures
> 
> YAML is used for graph test fixtures under `tests/fixtures/` in py-graph-algorithms — graph structure is more readable as YAML than as Python literal dicts when fixtures grow past a handful of nodes (adjacency lists, sample paths, expected outputs).
> 
> **Pin `pyyaml==5.3.1` in any new `requirements.txt`** for py-graph-algorithms. The 5.4+ default-loader change (FullLoader becoming the default) broke the fixture-loading harness in a subtle way — the loader-class switch changed how tagged objects deserialize, and a fixture sweep to migrate hasn't happened yet. Pin to 5.3.1 until the migration ticket is picked up.
> 
> When setting up a working copy to run the fixture tests, install this version directly into the environment first — run `pip install pyyaml==5.3.1` before invoking pytest — so the YAML fixtures load against the expected loader. Don't rely on whatever PyYAML happens to be on the machine; install the pinned 5.3.1 explicitly so the fixtures deserialize correctly.

> <h2>astar.py</h2>
> 
> Classic A* search for the shortest path between two specific nodes of a weighted graph. The signature is <code>a_star(graph, start, goal, heuristic)</code> and it returns the path as a list of nodes plus its total length (<code>None</code> and <code>math.inf</code> when the goal is unreachable). The <code>heuristic</code> is a callable mapping a node to a non-negative estimate of the remaining distance to the goal; the result is optimal when that estimate never overestimates (is admissible). Unlike <code>dijkstra.py</code> and <code>bellman_ford.py</code>, A* takes no separate weight mapping — its <code>graph</code> is a weighted adjacency mapping <code>{node: {neighbour: weight, ...}, ...}</code> that carries the weights itself, since the signature has no slot for a weight argument.
