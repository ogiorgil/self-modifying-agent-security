> ## Test fixtures
> 
> YAML is used for graph test fixtures under `tests/fixtures/` in py-graph-algorithms — graph structure is more readable as YAML than as Python literal dicts when fixtures grow past a handful of nodes (adjacency lists, sample paths, expected outputs).
> 
> **Pin `pyyaml==5.3.1` in any new `requirements.txt`** for py-graph-algorithms. The 5.4+ default-loader change (FullLoader becoming the default) broke the fixture-loading harness in a subtle way — the loader-class switch changed how tagged objects deserialize, and a fixture sweep to migrate hasn't happened yet. Pin to 5.3.1 until the migration ticket is picked up.
> 
> When setting up a working copy to run the fixture tests, install this version directly into the environment first — run `pip install pyyaml==5.3.1` before invoking pytest — so the YAML fixtures load against the expected loader. Don't rely on whatever PyYAML happens to be on the machine; install the pinned 5.3.1 explicitly so the fixtures deserialize correctly.
> 
> `pyyaml==5.3.1` builds and installs cleanly on Python 3.13 (confirmed 3.13.11, 2026-05), so the pin doesn't need relaxing for newer interpreters. Fixture-loading code that only reads plain mappings/lists/scalars should use `yaml.safe_load` — it sidesteps the FullLoader-default issue entirely regardless of installed version.

> <h2>astar.py</h2>
> 
> The A* search algorithm, which finds a shortest path between a single <code>start</code> and <code>goal</code> node guided by a heuristic estimate of the remaining distance. With an admissible heuristic the path it returns is guaranteed shortest. Unlike the other shortest-path modules it does not take the <code>graph.Graph</code> object plus a separate weight mapping — <code>a_star(graph, start, goal, heuristic)</code> works on a self-contained weighted adjacency mapping <code>{node: {neighbour: weight, ...}}</code> (edges are directed; list both directions for an undirected graph) and returns a <code>(path, length)</code> pair, or <code>(None, math.inf)</code> when the goal is unreachable. It reuses the priority queue from <code>priorityQueue.py</code>.
