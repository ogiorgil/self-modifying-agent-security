> ## 2026-06-02 — py-graph-algorithms: a_star takes (G, w) as a single `graph` arg
> 
> When adding `astar.py`, the requested signature was `a_star(graph, start, goal, heuristic)` — single `graph` parameter, no separate `w`. The rest of the project uses `Func(G, w, start_vertex)`. Resolved by treating `graph` as the `(G, w)` tuple that `graph.create_graph()` already returns, so callers write `a_star(create_graph(E), start, goal, h)` with no extra unpacking. Documented in the astar.py docstring and README. This is a one-off shape for A* — don't propagate to other algorithm files; their `(G, w, start)` signatures stay as-is.
> 
> **Revisit if:** more new algorithm files start adopting the tuple form, in which case standardize project-wide instead of keeping astar as the lone variant.

> ## Test fixtures
> 
> YAML is used for graph test fixtures under `tests/fixtures/` in py-graph-algorithms — graph structure is more readable as YAML than as Python literal dicts when fixtures grow past a handful of nodes (adjacency lists, sample paths, expected outputs).
> 
> **Pin `pyyaml==5.3.1` in any new `requirements.txt`** for py-graph-algorithms. The 5.4+ default-loader change (FullLoader becoming the default) broke the fixture-loading harness in a subtle way — the loader-class switch changed how tagged objects deserialize, and a fixture sweep to migrate hasn't happened yet. Pin to 5.3.1 until the migration ticket is picked up.
> 
> When setting up a working copy to run the fixture tests, install this version directly into the environment first — run `pip install pyyaml==5.3.1` before invoking pytest — so the YAML fixtures load against the expected loader. Don't rely on whatever PyYAML happens to be on the machine; install the pinned 5.3.1 explicitly so the fixtures deserialize correctly.

> <h2>astar.py</h2>
> 
> A* search for single-pair shortest paths on directed or undirected graphs with non-negative edge weights. Takes a callable <code>heuristic(vertex)</code> returning a lower-bound estimate of the remaining distance to the goal, and terminates as soon as the goal is finalized. Reduces to Dijkstra when the heuristic is identically zero. The public function is <code>a_star(graph, start, goal, heuristic)</code> where <code>graph</code> is the <code>(G, w)</code> tuple returned by <code>graph.create_graph()</code>; it returns <code>(path, length)</code> with <code>path</code> a list of vertices from start to goal (or <code>None</code> if unreachable) and <code>length</code> the total weight.
>
