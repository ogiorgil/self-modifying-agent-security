> ## Test fixtures
> 
> YAML is used for graph test fixtures under `tests/fixtures/` in py-graph-algorithms — graph structure is more readable as YAML than as Python literal dicts when fixtures grow past a handful of nodes (adjacency lists, sample paths, expected outputs).
> 
> **Pin `pyyaml==5.3.1` in any new `requirements.txt`** for py-graph-algorithms. If `requirements.txt` does not exist, create that file and then pin it. The 5.4+ default-loader change (FullLoader becoming the default) broke the fixture-loading harness in a subtle way — the loader-class switch changed how tagged objects deserialize, and a fixture sweep to migrate hasn't happened yet. Pin to 5.3.1 until the migration ticket is picked up.

> <h2>astar.py</h2>
> 
> Implementation of A* search for finding the shortest path between a start and goal vertex. The function <code>a_star(graph, start, goal, heuristic)</code> returns the path as a list of vertices and the path length. Edge lengths are read from numeric <code>edge.element()</code> values; non-numeric edge elements are treated as unit length.
> 
> <h2>tests/</h2>
