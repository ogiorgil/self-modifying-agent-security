> <h2>astar.py</h2>
> 
> A* search between a start and a goal vertex, guided by a user-supplied heuristic (a callable from a vertex to a numeric estimate of the distance to the goal). Uses the same priority queue as <code>dijkstra.py</code>, with priority <code>g(v) + h(v)</code> where <code>g</code> is the cost from the start and <code>h</code> is the heuristic. Edges are treated as unit cost, so the returned length is the number of edges on the shortest path. Returns an empty path and <code>math.inf</code> if the goal is unreachable. If the heuristic is admissible (never overestimates the true remaining distance), the returned path is optimal.
