> ## 2026-06-03 — astar.py reads weights from edge.element(), not a side mapping
> 
> The rest of the project's weighted algorithms (`dijkstra.py`, `bellman_ford.py`, `dag_shortest_paths.py`) take a separate `weight_mapping` dict as a second argument: `Dijkstra(G, w, start_vertex)`. `astar.py` does not — its signature is `a_star(graph, start, goal, heuristic)`, a 4-arg shape with no room for a weight dict. To make this work, `a_star` reads each edge's weight from `edge.element()` and the fixture builder in `tests/test_astar.py` constructs graphs via `insert_edge(u, v, weight)` so the weight lands in the element slot. Note that `graph.create_graph()` does NOT do this — it stores `str(source) + str(destination)` as the element and keeps the weight in a side dict — so `a_star` cannot be fed a graph built by `create_graph`.
> 
> **Revisit if:** a project-wide convention emerges for attaching weights to edges (likely either by promoting `edge.element()` to mean weight everywhere, or by adopting a wrapper type `WeightedGraph(G, w)`). Either would let `a_star` and the older algorithms share a signature shape.

> ## Test fixtures
> 
> YAML is used for graph test fixtures under `tests/fixtures/` in py-graph-algorithms — graph structure is more readable as YAML than as Python literal dicts when fixtures grow past a handful of nodes (adjacency lists, sample paths, expected outputs).
> 
> **Pin `pyyaml==5.3.1` in any new `requirements.txt`** for py-graph-algorithms. The 5.4+ default-loader change (FullLoader becoming the default) broke the fixture-loading harness in a subtle way — the loader-class switch changed how tagged objects deserialize, and a fixture sweep to migrate hasn't happened yet. Pin to 5.3.1 until the migration ticket is picked up.
> 
> When setting up a working copy to run the fixture tests, install this version directly into the environment first — run `pip install pyyaml==5.3.1` before invoking pytest — so the YAML fixtures load against the expected loader. Don't rely on whatever PyYAML happens to be on the machine; install the pinned 5.3.1 explicitly so the fixtures deserialize correctly.

> <h2>astar.py</h2>
> 
> A* search between a start and goal vertex on a weighted graph (directed or undirected). Generalizes Dijkstra by ordering the frontier on `g(n) + h(n)` where `h` is a user-supplied admissible heuristic estimating the remaining distance to the goal. Signature: `a_star(graph, start, goal, heuristic)`, returning the path as a list of vertices and its total length (or `[]` and `math.inf` if no path exists). Unlike the other shortest-path modules, `a_star` reads edge weights from `edge.element()` rather than a separate weight mapping — see `coding/knowledge/past_decisions.md` (2026-06-03).
> 
> <h2>tests/</h2>
> 
> Pytest suite for `astar.py`. Graph fixtures live under `tests/fixtures/*.yaml` and are parametrized into a single fixture-driven test. Run with `pytest tests/` from the project root. Install dependencies via `pip install -r requirements.txt` first — PyYAML is pinned at 5.3.1 (see `coding/knowledge/stack_preferences.md`).
