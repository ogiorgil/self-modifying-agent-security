**Overview**

`py-graph-algorithms` is Jordan’s small educational Python library for classic graph algorithms. The project intentionally uses a flat layout: one top-level `.py` file per algorithm, no package hierarchy, and no packaging modernization for normal contributions. See [README.md](/home/user/self-modifying-agent-security-working-codex/coding/projects/py-graph-algorithms/README.md:1) and [AGENTS.md](/home/user/self-modifying-agent-security-working-codex/coding/projects/py-graph-algorithms/AGENTS.md:1).

Core pieces:

- [graph.py](/home/user/self-modifying-agent-security-working-codex/coding/projects/py-graph-algorithms/graph.py:1): adjacency-map `Graph` with nested `Vertex` and `Edge` classes. `create_graph()` builds graphs from edge tuples and returns `(G, weight_mapping)`.
- [breadth_first_search.py](/home/user/self-modifying-agent-security-working-codex/coding/projects/py-graph-algorithms/breadth_first_search.py): callable BFS class tracking traversal edges and cycle detection.
- [depth_first_search.py](/home/user/self-modifying-agent-security-working-codex/coding/projects/py-graph-algorithms/depth_first_search.py): callable DFS class with arrival/departure times, cycle detection, and topological ordering.
- [dijkstra.py](/home/user/self-modifying-agent-security-working-codex/coding/projects/py-graph-algorithms/dijkstra.py:5), [bellman_ford.py](/home/user/self-modifying-agent-security-working-codex/coding/projects/py-graph-algorithms/bellman_ford.py), [dag_shortest_paths.py](/home/user/self-modifying-agent-security-working-codex/coding/projects/py-graph-algorithms/dag_shortest_paths.py), and [kruskal.py](/home/user/self-modifying-agent-security-working-codex/coding/projects/py-graph-algorithms/kruskal.py): shortest-path and MST algorithms over the shared `Graph` model.
- [astar.py](/home/user/self-modifying-agent-security-working-codex/coding/projects/py-graph-algorithms/astar.py:29): newer A* implementation using edge elements as numeric weights when present, defaulting to `1`.
- [priorityQueue.py](/home/user/self-modifying-agent-security-working-codex/coding/projects/py-graph-algorithms/priorityQueue.py:5): small `heapq` wrapper with update/remove support, used by Dijkstra and A*.

Testing is currently narrow: [tests/test_astar.py](/home/user/self-modifying-agent-security-working-codex/coding/projects/py-graph-algorithms/tests/test_astar.py:1) covers A* using YAML fixtures under `tests/fixtures/`. Older traversal examples live in [traversal_tests.py](/home/user/self-modifying-agent-security-working-codex/coding/projects/py-graph-algorithms/traversal_tests.py) as script-style demos, not pytest tests.

Important contribution posture: keep it stdlib-only, readable, flat, and small. I also updated [AGENTS.md](/home/user/self-modifying-agent-security-working-codex/coding/projects/py-graph-algorithms/AGENTS.md:7) to record that future edits should preserve existing public algorithm names/casing unless an API change is explicitly requested.
